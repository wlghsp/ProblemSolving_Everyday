# Two Pointer Greedy Strategy 증명

## 문제: Container With Most Water (LeetCode 11)

넓이 = `min(height[l], height[r]) × (r - l)`

거리는 계속 줄어들므로, **높이만 늘려야 더 큰 넓이를 만들 수 있다.**

---

## 핵심 질문

"폭이 줄어들잖아? 안 줄이는 게 나을 수도 있는 거 아닌가?"

**답: 아니다. 더 작은 높이의 포인터를 움직이는 게 손실을 최소화한다.**

---

## 수학적 증명

### 현재 상태
```
l < r, height[l] <= height[r]
현재 넓이 = height[l] × (r - l)
```

### 경우 1: r을 한 칸 이동 (l은 고정)
```
새 넓이 = min(height[l], height[r-1]) × (r - 1 - l)
```

**분석:**
- 폭: `(r - 1 - l)` ← **1씩 줄어듦**
- 높이: `min(height[l], height[r-1])`
  - height[r-1] < height[r]일 가능성이 높음
  - 높이도 같거나 더 작아질 가능성 높음
- **결론: 폭 감소 + 높이 감소 (최악의 경우)**

### 경우 2: l을 한 칸 이동 (r은 고정) ⭐ 선택
```
새 넓이 = min(height[l+1], height[r]) × (r - 1 - l)
```

**분석:**
- 폭: `(r - 1 - l)` ← **1씩 줄어듦** (경우 1과 동일)
- 높이: `min(height[l+1], height[r])` = `height[r]` (r이 더 크다고 가정했으므로)
  - height[l+1]이 height[l]보다 클 가능성이 있음
  - **높이가 나아질 수 있음!**
- **결론: 폭 감소하지만, 높이 이득 가능성 (최적)**

---

## 핵심 통찰

| 포인터 | 폭 변화 | 높이 변화 | 트레이드오프 |
|--------|--------|----------|-----------|
| r 이동 (큰 쪽) | -1 | 감소 가능성 높음 | 최악 |
| **l 이동 (작은 쪽)** | **-1** | **증가 가능성 있음** | **최적** |

**더 작은 높이의 포인터를 움직이는 것:**
- 폭 손실은 동일 (어차피 1씩 감소)
- 높이 이득 가능성을 최대화
- **손실을 최소화하는 greedy 전략**

---

## 코드 구현

```python
if height[l] <= height[r]:
    l += 1  # 더 작은 쪽을 움직임
else:
    r -= 1
```

---

## 왜 이게 최적인가?

**선택지:**
1. l을 움직인다 → 높이가 커질 수 있음 (최선)
2. r을 움직인다 → 높이가 작아질 가능성 (최악)

**따라서 작은 쪽을 움직여서 높이 개선 기회를 노린다!**

---

## 시간/공간 복잡도

- **시간**: O(N) - 각 포인터는 최대 N번 이동
- **공간**: O(1) - 상수 공간만 사용

이 greedy 접근법은 O(N²) 브루트포스를 O(N)으로 최적화한다.
