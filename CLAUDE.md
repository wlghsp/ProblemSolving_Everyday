# 프로젝트 작업 규칙

## 목적
**알고리즘 학습**을 위한 프로젝트입니다. 모든 파일은 사용자가 직접 풀고, 이해하고, 기억하는 것을 돕는 방향으로 만들어야 합니다. 답을 바로 주는 것이 아니라, 단계적으로 스스로 풀 수 있도록 유도하는 것이 핵심입니다.

## LeetCode 문제 파일 생성 규칙

### 폴더 구조
```
LeetCode/leetcode{번호}_{문제이름}/
├── Solution.py   # 풀이 코드 + 테스트 케이스
└── 힌트.md       # 3단계 힌트 문서
```

### Solution.py 작성 규칙
- 클래스와 메서드 시그니처만 제공 (pass)
- `if __name__ == "__main__":` 블록에 테스트 케이스 포함
- 연결 리스트 등 자료구조가 필요하면 helper 함수 포함
- 기존 폴더가 있으면 Solution.py로 파일명을 다르게 해서 추가

### 힌트.md 작성 규칙 (3단계 힌트 구조)

상단에 문제 요약, 예시, 제약 조건을 배치한 후, 힌트를 3단계로 나눠서 작성합니다.

**Level 1 - 방향 힌트** (먼저 이것만 보고 시도)
- 어떤 알고리즘/자료구조를 써야 하는지만 알려줌
- 핵심 아이디어 1~2줄

**Level 2 - 구조 힌트** (Level 1로 안 되면)
- 주석만 있는 빈 코드 (의사 코드)
- 어떤 순서로 무엇을 해야 하는지 골격만 제공

**Level 3 - 전체 풀이** (복습/확인용)
- 코드 블록 안에 주석으로 상세 설명
- 자주하는 실수, 관련 문제 포함


### 기타 규칙
- 여러 풀이 방법이 있으면 비교표 포함
- 내용이 너무 길어지면 별도 가이드 문서로 분리 (예: 병합정렬_가이드.md)
- 풀이 파일: Python / 힌트 문서: 한국어
